# Product Design Guide

This app is a mobile-first monitoring tool. Optimize for repeat checking, not browsing.

## Layout

- One primary action per screen.
- Keep filters on the surfaces where the user makes the decision: dashboard and watchlist.
- Keep admin locked until credentials are entered.
- Do not reuse admin as a home for general workflow controls.

## Information Density

- Show the compact summary first.
- Reveal the detailed evidence only after a click.
- Keep score, price, and source labels separated.
- Remove duplicate labels that repeat the same fact in different sections.

## Color And Contrast

- Do not introduce black boxes just to hold forms or status lines.
- Keep CTA text readable on the live background.
- Use one shared tab style across the app, including admin.
- If a panel changes theme, the text and hover states must still match the rest of the app.

## Copy

- Short is better than descriptive when the action is obvious.
- Avoid internal labels in public UI.
- Prefer plain Korean over stacked jargon.
- Use source labels only when they help the user decide.
